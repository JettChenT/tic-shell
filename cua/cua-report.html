CUA CLI Implementation Report

A Rust command-line interface for computer-use automation on Wayland, implemented niri-first with compositor-rendered window screenshots and uinput input injection.

Status

Rust crate builds Patched niri active Silent screenshots verified Workspace describe verified Type command not live-tested into shell

The CLI lives in /home/jettc/osdev/tic-shell/cua. It targets niri first and discovers the active compositor socket from NIRI_SOCKET, XDG_RUNTIME_DIR, or matching /run/user/*/niri.*.sock sockets.

The running compositor is the patched forked niri at /nix/store/00lrnrsrms4k2595q8ayvj1n50kqlh2i-niri-26.04-cua/bin/niri. Its IPC command screenshot-window now accepts --notify false.

Architecture

Metadata

  • niri msg --json workspaces resolves active and requested workspaces.
  • niri msg --json windows lists windows, titles, app IDs, PIDs, focus state, and layout sizes.
  • CUA_WORKSPACE_ID or NIRI_WORKSPACE_ID can supply the default workspace.

Screenshots

  • cua asks niri to render a window by ID directly from compositor state.
  • The command passes --notify false to avoid desktop notifications.
  • Window screenshots are written to temp files and combined into a workspace composite PNG.

Input

  • click, scroll, and type-text use Linux uinput.
  • Input is delivered through regular focused-seat behavior; background clicks are not supported by Wayland/niri.
  • Click and scroll focus the target window before calculating absolute output coordinates.

Fallback

  • --intrusive-fallback remains available for grim-based screenshots after focusing a window.
  • The normal path no longer uses grim and does not shift focus for screenshot capture.

Command Interface

Command Signature Behavior
describe-workspace
describe_workspace		 cua describe-workspace [workspace_id] Lists windows in a workspace, captures each window, creates a composite screenshot, and prints JSON.
screenshot-window
screenshot_window		 cua screenshot-window <window_id> Captures a single niri window by ID to a temp PNG using non-intrusive niri IPC.
click cua click <window_id> <x> <y> Focuses the target window, moves the virtual pointer to window-relative coordinates, and sends a left click.
type-text
type		 cua type-text <window_id> <text> Focuses the target window and emits supported ASCII text through a virtual keyboard.
scroll cua scroll <window_id> <up|down|left|right> <amount> [x] [y] Focuses the target window, moves to an optional window-relative point, and sends wheel events.

Verification Notes

  • cargo check -p niri-ipc -p niri-config passed in the niri fork.
  • cargo build passed in cua/.
  • niri msg action screenshot-window --help shows --notify <NOTIFY>.
  • Generated screenshot files were written by the compositor user, and the workspace composite was written by the CLI caller.
The normal screenshot path is non-intrusive for focus and silent for notifications. Input commands still use normal focused input routing because Wayland/niri do not provide background click or multi-cursor semantics for arbitrary clients.
